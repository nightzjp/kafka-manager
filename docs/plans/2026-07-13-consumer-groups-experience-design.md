# Consumer Groups 体验重构设计

## 目标

让研发人员能快速定位高 Lag 消费组、判断影响范围、查看分区 Offset 进度，并在明确的危险操作区域安全重置 Offset。页面延续 Kafka Manager 现有深浅主题和数据密集型控制台风格。

## 列表页

- 页面顶部展示消费组总数、存在积压的消费组数、总 Lag 和异常分区数。
- 提供名称搜索、状态筛选和 Lag 排序；默认按总 Lag 从高到低排列。
- 使用诊断型表格代替当前松散列表，列出消费组、状态、成员数、分区数、总 Lag、Lag 强度和详情入口。
- 消费组名称是主要入口；整行提供稳定 Hover，但不依赖不可访问的整行点击。
- Lag 使用数字、状态文字和进度条共同表达，不只依赖颜色。

## 详情抽屉

- 桌面端从右侧打开 760–900px 宽抽屉，保留列表上下文；移动端转为全屏。
- 顶部显示消费组名称、状态、协议、成员数、总 Lag 和异常分区数。
- 分区表格按 Lag 从高到低排列，展示 Topic、Partition、当前 Offset、末端 Offset、Lag 和消费进度。
- 支持按 Topic/Partition 搜索和仅查看存在 Lag 的分区。
- Offset 重置放在底部独立危险区域，说明影响并保留输入消费组名称确认机制。
- 抽屉支持 Escape、背景滚动锁定、焦点陷阱和关闭后焦点恢复。

## 响应式与状态

- 1024px 以上使用完整表格和右侧抽屉。
- 768–1024px 隐藏次要列，抽屉占据大部分宽度。
- 768px 以下列表转为紧凑卡片，抽屉全屏，分区表格局部横向滚动。
- 加载、空结果、接口错误和重置进行中均提供明确反馈。

## 验证

- 单元测试覆盖 Lag 排序、汇总统计和分区过滤。
- TypeScript、Vitest、生产构建、Go 全量测试和 vet 全部通过。
- 使用真实 Consumer Group API 验证列表字段和空数组处理。
