# Kafka Manager 前端体验重构设计

## 1. 目标

把当前“功能可用但操作割裂”的控制台重构为研发人员可以长期日常使用的 Kafka 工作台。重构不改变后端能力边界，重点解决主题割裂、操作路径长、Topic 与消息脱节、JSON 阅读体验弱、响应式缺失和危险操作层级不明确。

## 2. 设计系统

设计基线保存于 `design-system/kafka-manager/MASTER.md`。采用数据密集、逐层钻取的专业控制台风格。

- 支持日间、夜间、跟随系统三种主题策略；手动选择写入浏览器本地存储。
- 深色使用 Slate/OLED 层级，浅色使用低饱和灰白层级。
- 主操作使用蓝色，健康使用绿色，警告使用琥珀色，危险使用红色。
- 正文使用 Fira Sans；JSON、Offset、Broker 地址等技术数据使用 Fira Code，并提供系统字体回退，部署不依赖外部字体加载。
- 统一使用内联 SVG 图标，不使用字符符号或 Emoji。
- 交互过渡为 150–200ms；遵循 `prefers-reduced-motion`。

## 3. 全局框架

桌面端使用可折叠左侧导航和顶部上下文栏。导航包含 Dashboard、Topics、消息检索、消费组、集群配置和操作审计。顶部栏负责集群切换、连接状态、主题切换和退出。

移动端不固定最小宽度，导航变为可开合抽屉，表格允许局部横向滚动而不是整个页面溢出。所有页面共享一致的内容宽度、间距、按钮、输入框、状态徽标、加载骨架、空状态和错误反馈。

## 4. Topic 工作区

Topic 列表保留搜索和创建，并增加健康状态、分区数、复制名称和“查看消息”快捷操作。点击行进入独立 Topic 工作区，而不是元数据弹窗。

工作区顶部显示面包屑、Topic 名称、健康摘要和快捷动作，包含四个标签：

1. 概览：分区、复制因子、ISR 异常、关键配置和分区分布。
2. 消息：自动绑定当前 Topic，支持有界查询、实时跟随和生产消息。
3. 分区：Leader、Replica、ISR、Offline Replica，并提供扩分区入口。
4. 配置：搜索当前配置、修改值、恢复默认值。

独立消息检索页面继续保留，作为跨 Topic 高级入口；从 Topic 工作区进入时不重复填写 Topic。

## 5. JSON 消息体验

消息默认尝试解析为 JSON。成功时提供语法着色、对象/数组折叠、全部展开/折叠、复制格式化 JSON、格式化/原始视图切换。解析失败时明确标记为 Text，不显示错误式的 JSON UI；不可打印数据标记为 Binary。

消息列表展示时间、Partition、Offset、Key 和单行内容预览。详情面板展示元数据、Key、Headers 和 Value。生产消息默认使用 JSON 编辑模式，发送前校验并定位错误；用户可切换纯文本模式。当前 Topic、Partition 等上下文自动继承。

## 6. Dashboard 与其他页面

Dashboard 按异常优先组织：顶部首先显示离线集群、ISR 异常和高 Lag，再展示 Broker、Topic、Partition 等规模指标。集群卡提供 Lag 趋势和直达 Topic/消费组入口。

消费组页面突出总 Lag、状态和问题分区，详情中集中放置 Offset 重置和删除等危险操作。配置页采用分组表单和连接类型联动。审计页使用可筛选表格/时间线并明确成功失败状态。

## 7. 可访问性与反馈

- 所有图标按钮提供可访问名称和 Tooltip。
- 表单标签与控件关联；键盘焦点始终可见。
- 弹窗支持 Escape、焦点管理和背景滚动锁定。
- 颜色不是状态的唯一表达，同时显示文字或图标。
- 保存、删除、发送等操作显示进行中、成功和失败反馈。
- 搜索输入使用延迟值，避免每次击键立即请求。

## 8. 技术实现

继续使用 React 18、TypeScript 和现有零运行时 UI 依赖方案。新增主题 Context、图标组件、Toast、Tabs、JSON Viewer 和 Topic Workspace 等内部组件，不引入大型组件库。路由仍由应用内部状态管理，但将 Topic 详情状态显式建模，支持从列表和消息快捷入口进入。

## 9. 验证

- 组件测试覆盖主题持久化、Topic 工作区导航、JSON 格式化/原始切换、JSON 生产校验和上下文继承。
- 前端类型检查、Vitest 和生产构建全部通过。
- 使用真实 Kafka 数据验证 Dashboard、Topic、消息和消费组主路径。
- 检查 375、768、1024、1440 宽度，深浅主题、键盘焦点和减少动态效果。
