# 工作区效率优化设计

## 目标

优化研发日常高频路径：桌面导航可折叠、Topic 修改后就地更新、消息过滤条件可通过 URL 恢复和分享。

## 侧栏折叠

- 首次访问默认展开，宽度保持 236px。
- 折叠后宽度 72px，仅显示品牌标识、菜单图标和集群健康灯。
- 当前菜单继续使用左侧蓝色状态条和背景高亮。
- 折叠菜单在鼠标悬浮和键盘聚焦时显示即时文本提示，同时保留 `aria-label` 与 `title`。
- 折叠按钮位于侧栏右侧边缘，使用方向箭头表达状态。
- 状态写入 `localStorage`，键名为 `kafka-manager.sidebar.collapsed`。
- 820px 以下保持现有抽屉菜单，忽略桌面折叠宽度，但不清除用户偏好。

## Topic 就地刷新

App 提供单个 Topic 的精确刷新函数，通过现有 Topic 搜索接口获取最新元数据。Topic 工作区提供手动刷新按钮；扩分区成功后等待最新元数据返回，再更新标题、标签数量和分区表，不要求返回列表。

配置列表仍使用自己的加载函数，在修改配置后立即刷新当前配置列表。

## 消息过滤 URL

过滤器从 `location.search` 初始化，并在 Key、Value、JSON 条件或扫描上限变化时使用 `history.replaceState` 写回当前 URL，不制造大量浏览器历史记录。

URL 复用 API 参数：`keyFilter`、`keyOperator`、`valueFilter`、`valueOperator`、`jsonFilters`、`scanLimit`。非法枚举、损坏 JSON 和越界扫描量使用安全默认值；离开消息页面时，App 的路径导航自然清除查询参数。

## 测试

- 侧栏偏好读取、写入和异常存储降级。
- 消息过滤 URL 的解析、规范化和往返。
- Topic 精确选择逻辑，确保同名搜索结果中只取完全匹配项。
- 全量前端、Go 测试、生产构建和运行时健康检查。
